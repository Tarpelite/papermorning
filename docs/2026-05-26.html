各位早上好，今天Paper Morning开播。 先说一篇让我觉得很有意思的工作。WBench是一个交互式视频世界模型的评估基准。听起来像是纯视觉生成的事情，跟我们做科学计算的好像有点远。但我仔细看了它的设计，发现它测量的五个维度——视频质量、场景一致性、交互遵循、物理合规性——其实跟我们在意的世界模型能力高度相关。我们做PDE求解、搞物理模拟，说到底也是在构建一个能够准确预测物理演化的世界模型。他们提出的“物理合规性”这个维度很直接，就是检验模型生成的视频是否遵循物理规律，比如物体运动是否合理、碰撞是否真实。这其实和我们验证神经网络算子输出的物理一致性是一个思路。不过他们用的是视频生成的方式，而我们更关心的是数值精度和方程满足程度。这个benchmark如果扩展到数值物理领域，可能会很有意思。 说完世界模型，接着看一个关于智能体协调的工作。Foundation Protocol提出了一个面向智能体社会的协作层。这个概念挺宏大的——他们要解决的是当大量AI智能体部署之后，如何让它们之间建立可靠的关系、组织多任务协作、交换价值。说实话，我一开始觉得这个有点太infra了，跟我们做科学计算的有什么关系？但仔细想想，我们正在训练越来越大的科学基础模型，未来可能需要成百上千个专业模型协同工作。比如一个 PDE solver agent 负责方程求解，一个代码生成agent负责写仿真代码，一个数据分析agent负责结果解释。它们之间怎么通信、怎么协调、怎么保证安全？这可能就是Foundation Protocol试图回答的问题。他们提出的图结构化协作层，给异构实体（agent、工具、资源、人、组织）提供统一交互框架的思路，或许可以借鉴到科学计算工作流的自动化设计上来。 接下来看一个更贴近交互界面的工作。Macaron-A2UI是给个人智能体配备生成式UI的能力。简单说，就是让智能体不只是输出文字，还能动态生成合适的控件、选项和交互界面来帮你完成任务。这让我想到一个问题：我们做科学计算的toolchain其实非常依赖静态的配置文件、脚本接口、命令行参数。用户跟这些工具的交互方式几十年没怎么变过。如果能把生成式UI的能力引入到科学软件中会怎样？想象一下，你告诉一个科学助手你要做一个参数扫描，它不仅帮你写代码，还能即时生成一个交互式的控制面板，让你可以直观地调整参数、观察结果。这可能离我们还有点远，但方向值得关注。 然后是强化学习的一篇。DVAO提出了动态方差自适应的优势优化方法，用来处理多奖励强化学习中的训练不稳定问题。他们指出了传统scalarization方法的两个痛点：Reward Combination产生的优势值方差过大导致训练爆炸，Advantage Combination依赖静态超参数且忽略目标间的相关性。我们的观众里做物理强化学习的朋友可能关心这个——比如用RL来控制等离子体、调节飞行器姿态、或者优化分子生成，这些问题往往涉及多个物理目标的平衡。这篇工作提供的动态调整机制，看起来比简单的加权求和更合理。不过我有点好奇的是，它在真实物理系统上的表现如何，毕竟仿真环境和真实环境的reward分布往往差异很大。 最后聊聊那篇关于原生多模态建模路线图的文章。他们把现有的多模态方案分成了late-fusion、mid-fusion、early-fusion这几类，强调要向“架构层面的内在融合”转型。这让我想到科学数据的多模态问题。遥感数据有光谱、时序、空间信息；基因数据有序列、结构、功能注释；流体模拟有速度场、压力场、密度场。我们怎么把这些异构模态真正融合到一个模型里，而不是简单地拼装encoder-decoder？这篇论文试图定义“架构nativity”的概念，给了一个思考框架。但我怀疑他们对科学应用场景考虑得不够——科学数据的特点是物理约束强、领域知识深，单纯的网络融合可能不够，还需要把物理先验嵌进去。 今天的五篇论文看起来比较发散，从世界模型评估到智能体协作，再到UI生成、强化学习和多模态架构。但如果把它们串起来看，有一个共同的脉络：AI系统正在从“单一工具”向“协作生态”演进。以前我们训练一个模型解决一个任务，现在我们在考虑多个模型、多智能体、多模态的协同工作。这个趋势对科学计算意味着什么？可能意味着未来的科学AI系统不再是单个的PDE solver或者foundation model，而是一个由专业化agent组成的协作网络，各司其职又相互配合。这会是我们接下来几年需要认真思考的问题。
